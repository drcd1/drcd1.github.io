stuff i've made

Here's a few things I've made along the way. Some of them are personal projects, some are things-that-were-used-by-actual-people, and others are project courses where I went above and beyond (and a bit outside the scope) of what was needed. twin pathtracers | c++ vs rust

This is a little experiment I'm making, comparing the performance of C++ and Rust by writing a toy pathtracer in both languages.

So far, I've found that it's more natural to write higher performance code in Rust. Using similar constructs, and writing basic textbok code, I've found that the code in Rust runs ~50-100% faster. I'm looking forward to keep working on this project, and I'm really curious to see what happens as I turn this into a fully fledged (bidirectional) pathtracer. Here's the C++ version and the Rust version.

cloth simulation | "numerical methods for odes" project @ University of Saarland

This was a project for the course "Numerical Methods for ODEs" at the University of Saarland.

Since we had free reign over the project, (i.e.: do anything that can be described by an ordinary differential equation), I chose to do something a little more complicated and implement a simple mass-spring cloth simulation, with a few collision primitives, and tested it for (almost) all integrators covered in the lecture (which was really easy to do since I implemented the proper abstractions). You may find the code here, as well as a short report here.

ricochet robots instance visualizer | course material for ai @ Instituto Superior Técnico

A vizualizer used by the students in the Artifical Intelligence course at Instituto Superior Técnico in the year 2020/2021, created by me and Margarida Ferreira. It allowed students to create their own instances of Ricochet Robots, to play the game, save the solutions and playback any solution. Code can be found here, and you can play around with it here.

pathtracer | project for "3d programming" class @ Instituto Superior Técnico

Pathtracer done with Daniel Ramos for a class. The project was only required to be a simple ray tracer with some distributed ray-tracing elements (soft-shadows, glossy reflections, etc...), but we decided to implement a full pathtracer with support for HRD environment maps, normal maps, physically based shading, and bokeh depth of field, among other things. c++ renderer/rasterizer

This is a software rasterizer and renderer that I did to learn the basics of Computer Graphics, before I had ever seen a graphics API in my life. It's not very fast, but it has a few elegantly designed abstractions. It did at some point support shadow volumes (which is probably my favorite tech in real time rendering), but it was buggy and slow, so they are not there anymore.

stuff i've made

Here's a few things I've made along the way. Some of them are personal projects, some are things-that-were-used-by-actual-people, and others are project courses where I went above and beyond (and a bit outside the scope) of what was needed. Source is be available for some of them, so feel free to check it out!

You can also check out my shadertoy for a few small coding experiments I make every once in a while!

kurayami | bidirectional pathtracer

A toy renderer called kurayami that implements several advanced pathtracing algorithms, namely Vertex Connection and Merging (VCM) (Georgiev, I. et al, Hachisuka, T. et al)

It also implements some other advanced algorithms, such as path guiding in the style of (Müller et al, 2017), and a version of Metropolis Light Transport for VCM, in the style of (Šik et al, 2016)

This renderer served as the basis of my MSc. Thesis with the topic "MIS weights for Metropolised Bidirectional Rendering Algorithms", which is still under review. You'll be able to find the code here after the review is done.

real-time subsurface scattering

A little project to get the hang of WebGPU - it implements subsurface scattering and translucency. If your browser runs WebGPU (mine doesn't), you can find the impementation here.

various shadertoy mini-projects

Shadertoy is a wonderful little website that I use to randomly implement (simplified versions of) papers I've read or just test out little simple ideas. You can find my profile here.

cloth simulation | "numerical methods for odes" project @ University of Saarland

This was a project for the course "Numerical Methods for ODEs" at the University of Saarland.

Since we had free reign over the project, (i.e.: do anything that can be described by an ordinary differential equation), I chose to do something a little more complicated and implement a simple mass-spring cloth simulation, with a few collision primitives, and tested it for (almost) all integrators covered in the lecture (which was really easy to do since I implemented the proper abstractions). You may find the code here, as well as a short report here.

ricochet robots instance visualizer | course material for ai @ Instituto Superior Técnico

A vizualizer used by the students in the Artifical Intelligence course at Instituto Superior Técnico in the year 2020/2021, created by me and Margarida Ferreira. It allowed students to create their own instances of Ricochet Robots, to play the game, save the solutions and playback any solution. Code can be found here, and you can play around with it here.

ground truth ambient occlusion | godot 3.x

After fixing a couple of (very small) bugs in Godot, I thought it was time to try and contribute with something a little more ambitious, so I did an implementation of GTAO by Jimenez et al. I'm still working on it - as soon as it's done you'll be able to find the code here.

brushify | an "optimization" algorithm to paint pictures

I felt inspired enough by this tweet to create my own quick implementation of the algorithm described there. It truly was a pleasant surprise to see the results from what is effectively the simplest "optimization" algorithm. Code will be available soon.

older stuff

path tracer | project for "3d programming" class @ Instituto Superior Técnico

Path tracer done with Daniel Ramos for a class. The project was only required to be a simple ray tracer with some distributed ray tracing elements (soft-shadows, glossy reflections, etc...), but we decided to implement a full path tracer with support for HDR environment maps, normal maps, physically based shading, and bokeh depth of field, among other things. Code (not yet available) and Video

c++ renderer/rasterizer

This is a software rasterizer and renderer that I did to learn the basics of Computer Graphics, before I had ever seen a graphics API in my life. It's not very fast, but it has a few elegantly designed abstractions. It did at some point support shadow volumes (which is probably my favorite tech in real-time rendering), but it was buggy and slow, so they are not there anymore. Code here! (not yet available).

CPF Maker/Viewer | software to create and view cooperative path finding problems

To aid Margarida Ferreira in doing her project for solving cooperative path finding problems, I made a little editor in SFML to ease her creation of new instances of the problem. Together, we created a viewer so that we could easily visualize the solutions. The CPFMaker is here!, and the instance viewer is here

fractal renderer | written in sfml 2.0

A sort of learning project to figure out how to work with SFML and render fractals. It's not much, but it was sort of the gateway to learn more about fractals, SDFs and the demoscene. Code here! (not yet available)

cairo/gtk2 renderer

After I finished my first Programming course, I became aware of my love for Computer Graphics (which made me switch from Physics to Computer Science the next year). So naturally I used the tools I knew (cairo and gtk2) to create a simple renderer. Since cairo is a library for 2D drawing, this is incredibly inefficient. But it was really fun to do! Code here! (not yet available)

super disk collider! | Project for "Programming" course @ Instituto Superior Técnico

This was the project for the first Programming course I took (and my first time programming in general!). Me and Diogo Valada went above and beyond what was asked of us and instead of solving the problem of two discs colliding analytically, we created a relatively* robust collision solver that works for many disks. Please do not look at the code (not yet available).

*okaay, it's not guaranteed to terminate BUT when it does it yields correct chain collisions!
